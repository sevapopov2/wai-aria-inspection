Неправильное использование атрибута aria-hidden

Объяснение проблемы и пример

В спецификации WAI-ARIA существует атрибут "aria-hidden", предназначенный для скрытия элемента, не являющегося важным для пользователя программы экранного доступа. При использовании данного атрибута элемент не отображается в дереве API доступности, что делает его невидимым для скринридера, однако зрячий пользователь сможет без затруднений взаимодействовать с данным элементом.



В данном примере атрибут aria-hidden применён ко всему div региону, что полностью скрывает все его элементы от пользователя программы экранного доступа. Пользователь не сможет переместиться к редакторам или кнопке при помощи клавиш быстрой навигации, при навигации с помощью клавиши Tab он услышит слово "Пусто".

Исходный код примера

        
            <div id="aria-hidden-example" aria-hidden="true">
                <label for="yourname">Ваше имя</label>
                <input type="text" id="yourname" name="yourname">
                <br>
                <label for="youremail">Ваш email</label>
                <input type="email" id="youremail" name="youremail">
                <br>
                <button type="submit">Отправить</button>
            </div>

Возможное решение проблемы

Атрибут "aria-hidden" стоит применять на элементах, которые не имеют значения для пользователя программы экранного доступа. Это могут быть декоративные изображения, повторяющийся текст. Данный атрибут ненужно применять на фокуссируемых элементах (кнопки, ссылки, флажки, редакторы, радиокнопки).

Существуют примеры, в которых данный атрибут используется ошибочно, скрывая важные для пользователя элементы и даже области. Такой пример можно встретить в системе ведения отчетов об ошибках Jira При редактировании созданного отчета об ошибке на всей области редактирования установлен атрибут "aria-hidden", что не позволяет незрячему пользователю перейти к редактированию отчета.
