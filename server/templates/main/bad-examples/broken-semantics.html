Исправление семантики при помощи ARIA

Пример и объяснение проблемы

Плохим примером использования ARIA является применение атрибута роли к элементу вместо нативного тега HTML, с которым ассистивные технологии могут поумолчанию корректно взаимодействовать. Например, вместо использования HTML-тега <a> навешивать роль ссылки на span элемент. В данном случае программа экранного доступа определит, что это ссылка, но она не сможет корректно взаимодействовать с ней. Она не будет фокусируема при нажатии клавиши tab, для работы такой ссылки нужен отдельный самостоятельный обработчик данной ссылки, написанный на Javascript, также некоторые ассистивные технологии не будут адекватно взаимодействовать с такого рода ссылкой, так как они не могут адекватно работать с ARIA. Ниже приведен пример таких объектов.

Яндекс Find with tab key and click me

Исходный код примера

Ниже представлен исходный код примера.

    
        <div id="broken-semantics-example">
            <span id="broken-yandex-link" role="link">Яндекс</span>
            <span id="broken-button-example" role="button">Find with tab key and click me</span>
        </div>

Возможное решение проблемы

Решением данной проблемы является замена роли ARIA на соответствующий стандартный HTML-тег. ARIA не предназначена для исправления элементов, не имеющих семантики. Стоит помнить главное правило стандарта: не использовать ARIA тогда, когда это не является необходимостью. Неправильное использование стандарта может только навредить доступности.
