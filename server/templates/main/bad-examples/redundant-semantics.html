Добавление излишней семантики

Пример и объяснение проблемы

Существует практика, когда разработчики добавляют излишнюю семантику к элементу в надежде, что это улучшит доступность. Например, добавление роли ARIA к элементу в случае, когда уже имеется нативный HTML-тег с данной ролью, сообщающий ассистивным технологиям правильную семантику.

О проекте

Исходный код примера

    
        <div id="redundant-semantics-example">
            <a href="{% url 'main:index' %}" id="redundant-link" role="link">О проекте</a>
            <button id="redundant-button" type="button" role="button">Click me!</button>
            <form role="form">
                <label for="day">День</label>
                <input id="redundant-date" role="combobox" type="date">
            </form>
        </div>

Возможное решение проблемы

Несмотря на то, что в данном случае программы экранного доступа работают стандартно с такими элементами, добавлять дополнительную роль ARIA нет необходимости, так как HTML уже делает это с помощью соответствующих тегов. Также это может некорректно работать со старыми версиями программ экранного доступа.
